# アプリケーション名
ShoppingDays

# アプリケーション概要
予定管理用のカレンダー機能と、共有可能な買い物リスト機能を搭載したアプリケーションです。  
スケジュールと買い物リストは互いに連携せず、独立して機能します。  
リストを共有することで、購入が必要な物品の情報が共有者に伝達され、買い忘れを防止する効果を提供します。

# 利用方法
このアプリケーションの基本的な利用方法は以下の通りです。

・トップページから「ユーザー新規登録」画面に移動し、登録を行う  
※登録完了後は自動的にログインされ、カレンダーページへ遷移します  
・「ログアウト」リンクからログアウト可能  
・「ログイン」リンクからログイン可能  
・カレンダー内の日付セルに表示される「＋」マークをクリックして、買い物リストを作成  
・買い物リストの詳細画面でアイテム（品物）の追加・編集・削除が可能  
・カレンダー画面右上の「新しい予定を作成」ボタンから「予定」を作成  
・予定の詳細画面で予定の編集・削除が可能  

# アプリケーションを作成した背景
日常生活の中で、スーパーやコンビニに買い物へ行った際に、家族から頼まれた日用品や食料品をつい買い忘れてしまうことがありました。  
この「買い忘れ」を防ぐ仕組みがあれば便利だと考え、アプリケーションの制作に取り組みました。
対象とするのは、自分や家族をはじめ、日常的に買い物をする主婦・主夫などの生活者です。  
誰もが簡単に操作でき、手軽に買い物リストを管理できることで、買い忘れを防止できるアプリを目指しました。

# 実装した機能についての画像やGIFおよびその説明
- ユーザー管理機能  
 ・ユーザーログイン／ログアウト  
 ・ユーザー情報変更  
 ・ユーザー削除  

- カレンダー機能  
 ・カレンダー表示（Simple Calendar を利用）  
 
- 買い物リスト機能  
 ・買い物リストの表示・作成・編集・削除  
 ・品物の表示・追加・編集・削除  
 ・購入済みチェックによるステータス管理  

- スケジュール機能  
 ・スケジュールの表示・作成・編集・削除  

- 編集履歴参照機能  
 ・買い物リストの名前と日付の変更履歴表示  
 ・品物の追加・編集・削除の履歴表示  

- フィルタ表示機能  
 ・編集履歴参照の項目（ユーザー・買い物リスト・分類・アクション・日付）をフィルタ表示する  

- 通知機能  
 ・「今日の買い物リスト」と「今日の予定」がある場合、ログイン時にバナー通知を表示  
 ・ 通知の ON/OFF 切り替え可能

# 改善点
1. ユーザーごとの予定の色分け機能  
- 目的: カレンダー上で誰の予定かが視覚的に分かるようにする。  
 ・users テーブルへの color カラムの追加。  
 ・新規ユーザー登録時の色自動割り当てロジック（before_create コールバック）。  
 ・カレンダービューでの色情報の適用（HTML/CSS）。  

2. カレンダーへの六曜表示機能の追加  
- 目的: カレンダーに日本の六曜を表示し、視認性を高める。  
 ・六曜計算用のRuby gem（例：Qreki）の導入。  
 ・Simple Calendarのビューファイルでの表示ロジックの実装。  
 ・六曜ごとの装飾用CSS。  

3. 予定に対するファイル添付機能  
- 目的: 予定に関連する資料（PDF、画像など）を添付・管理する。  
 ・Rails標準の Active Storage の導入。  
 ・Plan モデルと添付ファイルの関連付け。  
 ・コントローラでのファイルパラメーターの許可。  
 ・フォームと詳細ビューにおけるアップロード/ダウンロード機能の実装。  

4. 時刻ベース通知機能の導入  
- 目的: 毎朝9時など、特定の時刻に今日の予定を自動通知する。  
 ・Notification モデル（データベース）の導入。  
 ・ユーザーと通知の1対多の関連付け。  
 ・外部のジョブスケジューラサービス（Cron、Google Cloud Schedulerなど）の利用。  
 ・通知手段（Firebase Cloud Messagingなど）の選定と導入。
